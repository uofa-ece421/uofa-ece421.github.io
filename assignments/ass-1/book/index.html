Assignment #2

Rustdoc and Cargo

The git repository for this assignment contains a simple library that provides a common interface to a couple of different serial communication protocols. The library worked well enough for our project, so we’re thinking of publishing it as a crate. However, there is a bit of work to be done before we can consider the library product ready.

In this assignment you will work with the rust tools cargo and rustdoc to refactor and document the library so that it is fit for release as a crate. The Rust book mentions both, but you can find detailed descriptions of the tools here:

  • The Cargo Book: https://doc.rust-lang.org/cargo/
  • The Rustdoc Book: https://doc.rust-lang.org/rustdoc/what-is-rustdoc.html

We’ll approach the task in three steps:

  1. (20 marks) Currently the entire library is in one file, lib.rs. Refactor the library into two modules: a uart module and a spi module.

  2. (30 marks) Write two example programs, demonstrating the use of the modules. For the uart example you can use “/dev/ttyUSB0” as a typical path; for spi “/dev/spi0.0” is a reasonable default. Your example programs should build, but they don’t have to run without error (since you likely don’t have an accessible SPI or UART device on your computer).

  3. (50 marks) Follow the Rustdoc book guidelines to document the crate. The rocket crate is a good example of what a properly documented crate looks like. While our simple serialio crate is not as complicated as rocket, its documentation should still have the same features of crate/module level and method/trait documentation formats, dependencies, and usage examples. The README can serve as the cover page for the crate, similar to what is in crates.io.

Notes:

  • Your solution will be graded by running cargo on the main branch of your repository, and checking the generated results. Therefore, you SHOULD push your local changes to the github repository, but you SHOULD NOT check in or push the generated results.
