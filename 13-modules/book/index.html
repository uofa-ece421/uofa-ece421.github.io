Kernel Modules

Kernel Modules

A kernel module is like a dynamically linked library for the kernel. Modules can be loaded and unloaded while the OS is running, and they can also be built into the kernel image (like a statically linked library).

Device drivers are popular to write as modules. They allow a generic kernel to run on many different hardware configurations by dynamically loading only the modules for the devices on the particular machine the kernel is running on.

  • insmod/rmmod - insert and remove a module from a running kernel

  • modprobe - a more intelligent tool to insert and remove modules. modprobe can search for a module given a name, and will automatically load any prerequisite modules that the module depends on.

  • signing - most systems require superuser privileges to insert and remove modules. Most systems also require that modules are cryptographically signed.

On MacOs, modules are called kernel extensions (kexts). MacOs also provides a tools to work with its modules: kextutil, kextstat, etc.
